# Nomenclatura

## Regra Geral

A nomenclatura técnica do projeto deve ser em inglês.

Isso inclui:

- Funções.
- Classes.
- Interfaces.
- Tipos.
- Tabelas.
- Colunas.
- Arquivos técnicos.
- Pastas de código.

## Documentação

A documentação de produto, arquitetura, fluxos e agentes deve ser escrita em português do Brasil.
